[회의 목적]
고객 설문 데이터를 수집/분석하여 자동으로 요약 리포트를 생성하는 기능을 2주 내 MVP로 완성한다.

[주요 논의 내용]
1) 데이터 수집 범위
- 설문 응답은 JSON 파일로 수집하며, daily batch로 저장한다.
- 응답 항목: id, category, text, score, timestamp
- 익명화: 이메일/전화번호 등 개인정보는 저장 전 마스킹 처리 필요

2) 분석 결과 구성(리포트)
- 전체 평균 점수, 긍정 비율(점수 4 이상), 카테고리별 통계(건수/평균/긍정비율)
- 키워드 추출은 1차 MVP에서는 간단히 빈도 기반으로 하고, 추후 LLM 기반으로 고도화한다.

3) API 및 화면
- 백엔드는 FastAPI로 /report 엔드포인트 제공
- 프론트는 리포트 카드 형태로 요약(한 줄 요약 + 지표 3개)과 상세(카테고리 표) 화면 제공
- MVP에서는 로그인 없이 내부 테스트 페이지로 제한

4) 일정/리스크
- 1주차: JSON 수집/정제 + 통계 산출
- 2주차: Gemini 요약 연동 + UI 표시 + QA
- 리스크: 스키마가 바뀌면 병합/분석 코드가 깨질 수 있으므로 Pydantic 검증을 넣는다.

[결정 사항]
- MVP 데이터 형식은 JSON으로 고정하고, 파일명 규칙은 YYYYMMDD_survey.json으로 한다.
- 점수 4 이상을 긍정으로 정의한다.
- 1차 릴리스에서는 키워드 추출은 빈도 기반으로 구현한다.
- FastAPI 엔드포인트는 /report 와 /health 두 개를 제공한다.
